RNN language model

发表于 5月 15日, 2021 分类于 Machine Learning ， nlp 阅读次数： Valine：

Lab3 RNN 语言模型，文本分类

实验目的

编写 RNN 的语言模型，并基于训练好的词向量，编写 RNN 模型用于文本分类。

这个实现需要编写一个 RNN 模型，并且基于训练好的词向量，使用编写的 RNN 模型完成文本分类的任务。

数据集和数据处理

数据集

这个实验使用的数据集为 aclImdb 数据集，这个数据集是一个二分类的情感分类数据集，数据集提供了 25000 个训练样本和 25000 个测试样本。数据集下载地址为 ：Sentiment Analysis (stanford.edu) 。

这个实验还使用了 GLoVe 预训练词向量数据，使用的是 glove.6B 的词向量，下载地址为 ：GloVe: Global Vectors for Word Representation (stanford.edu)

数据处理

首先把数据解压到自己定义的路径，并且把 aclImdb 数据读入到内存中。

创建 vocab

然后使用 torchtext 库对语料进行处理，创建模型使用到的词库 vocab ，在创建词库的过程中，使用 glove 预训练的词向量对词库的词进行编码，得到 Embedding Matrix 。代码如下：

值得注意的是，实验中的 Vocab 仅仅使用了训练集上的数据，并没有使用测试集上的数据，也就是说如果测试集中出现了训练集中没有的单词，则这个单词的 Embedding 向量会使用空向量。

实验设计

在所有的实验中，数据都需要经过一个 Embedding 的矩阵，这个矩阵的参数是由 vocab 的映射矩阵初始化而成的，在实验过程中设置为可更新的参数。

在实验中，由于数据经过了 padding ，所以数据经过 Embedding 之后，输入到 RNN 模型之前，需要先经过 pack_padded_sequence 操作，这样 RNN 才能识别出句子的实际长度为多长。

model_type layer_num train_acc val_acc test_acc
LSTM 1 0.9901 0.8860 0.8624

RNN 2 0.8031 0.7778 0.7794

结果解释

通过上面的实验结果，可以发现 LSTM 模型的实验结果明显好于 RNN 的结果。由于句子长度最长的可以达到一千多个单词，因此可见 LSTM 在长程依赖上要好于 RNN ，能够表现得更好。

加深层数，实验结果差不多，可能是由于即使加深了层数，利用的结果还是最后一层的最后一个隐变量的输出接上全连接层最后预测类别。因此结果提升不大。

总结

通过这个实验，学会了 torchtext 这个库的使用，学会了自然语言处理的一些操作。还体会到了 RNN 模型在自然语言处理上的强大。

汉明码

简介

汉明码是一种具有一位纠错能力的编码。使用汉明码编码生成的 n 位二进制串，如果传输过程中有一位的值发生了改变，则在接受端可以检测出接收到的结果不正确，并且能够找出具体是哪一位发生了改变。找出是哪一位发生改变后，就可以纠正过来得到正确的结果。但是当有超过一位发生改变时，汉明码就不具有纠错能力了。

构造

令 \(H\) 表示一个 \(l \times (2^l - 1)\) 的二进制矩阵。\(H\) 的每一列都有 \(l\) 位，并且 \(H\) 的第 \(i\) 列是十进制数 \(i\) 的二进制表示 \(i \in [1, 2, \dots, 2^l - 1]\) 。可以记 \(H = [h_1, h_2, \dots, h_{2^l - 1}]\) 其中 \(h_i\) 是 \(l\) 维的列向量。值得提及的是，\(h_i\) 不是零向量。以下j加法计算中，得到的结果都对 2 取模，即 \(010 + 110 = 100\) 。

计算输出：最后根据 \(\alpha\) 来计算输入信息的加权平均：\(a = \sum_{i=1}^N \alpha_i x_i\)
